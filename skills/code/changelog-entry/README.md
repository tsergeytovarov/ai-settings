# changelog-entry

Скилл обновляет корневой `CHANGELOG.md` проекта по формату Keep a Changelog на русском.

## Когда вызывается

- Явно: «добавь запись в changelog», «обнови changelog».
- Автоматически: после `feat:` или `fix:` коммита (user-visible изменения).

## Когда пропускается

- `chore:`, `docs:`, `test:`, `refactor:`, `style:`, `ci:`, `build:` коммиты — если они не видны пользователю.

## Использование

После feat/fix коммита скажи Клоду: «добавь запись в changelog». Он прочитает последний коммит, решит, в какую секцию `[Unreleased]` писать, и добавит одну строку.

## Правила записи

- Пиши то, что видит пользователь — не имена файлов и функций.
- Активный залог, русский язык.
- Одна строка — одно изменение.
- Только в `## [Unreleased]`. Релизные версии не трогаем.
