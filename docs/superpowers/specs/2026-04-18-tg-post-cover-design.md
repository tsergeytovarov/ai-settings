# Design Spec — обложки для tg-post-writer в стилистике Meridian

Дата: 2026-04-18
Статус: утверждён, готов к имплементации
Артефакт: `skills/work/tg-post-writer/` (в репозитории `ai-settings`)

## 1. Контекст

Скил `tg-post-writer` сейчас возвращает только текст поста. Задача — научить его параллельно генерировать **обложку 1080×1080 в визуальной стилистике Meridian**, которую можно прикрепить к посту в TG.

Визуальная стилистика заимствуется у OG-картинок Meridian (`ai-knowlege-base/frontend/src/app/**/opengraph-image.tsx`), но адаптируется под квадратный формат и минималистичный контент (только заголовок).

## 2. Проектные решения (все утверждены)

| Вопрос | Решение |
|---|---|
| Где рендерится PNG | Локальный Node-скрипт внутри скила (не Next.js route) |
| Формат | 1080×1080, квадрат |
| Что на картинке | Только title + визуальная рамка бренда |
| Kicker / таксономия | Нет |
| Подпись канала / домен | Нет |
| Шрифт | Literata SemiBold (бандлится в скил) |

Обложка — **опциональный** шаг: скил спрашивает пользователя, нужна ли она. Для коротких заметок (20-100 символов) обложка не предлагается вообще.

## 3. Визуальная спецификация

### 3.1 Полотно

- Размер: 1080×1080 px
- Фон: `#F7F4EE` (тёплый нейтрал Meridian)
- Формат вывода: PNG
- Padding safe-area: 88px со всех сторон

### 3.2 Палитра (все значения — 1-в-1 из OG Meridian)

| Токен | HEX | Назначение |
|---|---|---|
| `bg` | `#F7F4EE` | Фон |
| `accent` | `#C0632A` | Левая полоска + точки-акценты |
| `text-primary` | `#1A1510` | Title |
| `text-muted` | `#6B6258` | (резерв на будущее) |
| `text-subtle` | `#9A8F84` | (резерв на будущее) |

### 3.3 Композиция

Три константных элемента образуют «рамку» бренда:

**Левая полоска**
- `position: absolute; left: 0; top: 0`
- `width: 8px; height: 100%`
- `background: #C0632A`
- Отличие от OG (где 6px): квадрат 1080 крупнее, полоска 6px визуально теряется — поднимаем до 8px.

**Title-блок**
- `position: absolute; top: 120px; left: 88px; right: 88px`
- Литерата, см. §3.4
- `color: #1A1510`
- `text-align: left`

**Нижние точки-акценты**
- `position: absolute; bottom: 88px; right: 88px`
- Контейнер: `display: flex; align-items: center; gap: 20px`
- Первая точка: ø24, `background: #C0632A; opacity: 0.25; border-radius: 50%`
- Вторая точка: ø36, `background: #C0632A; border-radius: 50%`

Цитируется композиция нижнего ряда main-OG. Без домена и слова «Meridian» — остаётся только графический ритм.

### 3.4 Типографика title

- Шрифт: **Literata**, weight 600 (SemiBold)
- Стиль начертания: обычный (не italic)
- Цвет: `#1A1510`
- `letter-spacing: -1px`
- `line-height: 1.08`
- `text-align: left`

**Авто-скейл по длине** (длина считается в Unicode code points):

| Длина title (симв) | fontSize | maxLines |
|---|---|---|
| ≤ 30 | 128 | 2 |
| 31–60 | 104 | 3 |
| 61–100 | 80 | 4 |
| 101–140 | 64 | 5 |
| > 140 | обрезка до 139 симв + `…` | — |

Обрезка на границе слова (не посреди слова). Если в 139 символах нет пробела — режем жёстко по 140.

## 4. Технический пайплайн

### 4.1 Стек

- **satori** — JSX → SVG (тот же движок, что в `next/og`, но без edge-обёртки)
- **@resvg/resvg-js** — SVG → PNG, нативный биндинг, быстрый
- TypeScript, запуск через `tsx`

Не используем `@vercel/og` — это обёртка для edge-runtime, в CLI-сценарии лишняя.

### 4.2 Структура в скиле

```
skills/work/tg-post-writer/
├── SKILL.md                   (правим)
├── CHANGELOG.md               (обновляем версию)
├── references/
│   └── style-guide.md         (без изменений)
├── assets/
│   └── fonts/
│       └── Literata-SemiBold.ttf
└── tools/
    ├── package.json           (локальные deps)
    ├── tsconfig.json
    └── render-cover.ts        (основной скрипт)
```

### 4.3 Шрифт

- Файл: `Literata-SemiBold.ttf`, начертание 600
- Источник: Google Fonts, SIL Open Font License 1.1
- Требование: подмножество должно содержать кириллицу + латиницу + базовую пунктуацию
- Если полный файл слишком тяжёлый (>2 MB) — делаем `subset` через `pyftsubset` или `fonttools` на кириллицу+латиницу, без hinting

### 4.4 CLI-контракт

```
npx tsx render-cover.ts \
  --title "<строка 3-8 слов>" \
  --out /tmp/meridian-tg-<ISO-date>-<slug>.png
```

Флаги:
- `--title` (required): текст title
- `--out` (required): абсолютный путь для записи PNG

Поведение:
- Exit 0 — успех. На stdout: абсолютный путь к PNG.
- Exit 1 — ошибка валидации (title пуст, out невалиден). На stderr: человеко-читаемое сообщение.
- Exit 2 — ошибка рендера. На stderr: stack.

### 4.5 Выходной файл

- Имя по умолчанию: `/tmp/meridian-tg-{YYYYMMDD-HHMMSS}-{slug}.png`
- `slug` — первые 3-5 слов title, приведённые к lowercase, транслитерация кириллицы по BGN/PCGN-табличке, всё не-`[a-z0-9]` заменяется на `-`, дубли `-` сливаются, максимум 40 символов. Если после очистки пусто — используется `cover`.
- Формат: PNG, 1080×1080, без альфа-канала (RGB)

## 5. Интеграция в SKILL.md

### 5.1 Новый шаг в процессе

После шага 7 (pre-send checklist) добавляется шаг 8:

> **8. Предложить обложку.**
> Если пост длиннее 100 символов (то есть не short note по классификации `style-guide.md` §7) — спросить пользователя: «Сгенерить обложку для поста?». Если да:
> 1. Сформулировать `cover_title` (3-8 слов, тезис поста, см. §5.2).
> 2. Показать `cover_title` пользователю: «Обложка будет с заголовком: `<title>`. Так ок или поправить?». Пользователь может утвердить, попросить переделать или дать свой вариант.
> 3. После подтверждения — вызвать `tools/render-cover.ts` с `--title`.
> 4. Вернуть путь к PNG пользователю (с `file://` для быстрого открытия).
> Если пост короче 100 символов — обложка не предлагается.

Порог 100 символов — не случайный: по `style-guide.md` §7 это верхняя граница «short note», постов на 1-2 строки. На таких картинка избыточна, текст и так помещается в один экран ленты.

### 5.2 Правила формулировки `cover_title`

Title для обложки формулируется отдельно от текста поста. В тексте поста title не используется — живёт только на картинке.

**Формальные требования:**
- 3-8 слов
- не предложение, не хук-фраза — **тезис**
- третье лицо, нейтральный падеж (не «Смотрите сами...», не «Я думаю...»)
- первая буква — заглавная, остальные — как в обычном предложении
- без завершающих знаков (`.`, `!`, `?`, `…`)
- без кавычек (правило Meridian: кавычки только для прямой речи)
- без эмоджи, без хэштегов

**Примеры (для калибровки):**

| Жанр поста | cover_title |
|---|---|
| Новость про модель | `Claude 4.7: тихий апгрейд в reasoning` |
| Размышление про найм | `AI ломает воронку найма` |
| Разбор инструмента | `Cursor больше не единственный выбор` |
| Опыт / ретроспектива | `Год на AI-агентах: что сломалось` |

**Анти-примеры (так не делать):**

- «Смотрите сами, Claude 4.7 умнее Opus» — хук-фраза, не тезис
- «Что я думаю про AI-найм» — первое лицо, пустой смысл
- «Claude 4.7 — это новый король reasoning!» — восклицательный, кавычки-«это»
- «Новая модель от Anthropic доступна в API с ценой 3/15 долларов за миллион токенов и контекстом 1 миллион» — не тезис, а предложение-простыня

## 6. Тестирование

### 6.1 Ручные проверки (golden path)

Набор тест-кейсов на разные длины title. Для каждого — визуальный просмотр PNG:

1. `"Короткий тезис"` (14 симв) — fontSize 128
2. `"Claude 4.7: тихий апгрейд в reasoning"` (37 симв) — fontSize 104
3. `"AI ломает воронку найма, и вот почему это надолго"` (50 симв) — fontSize 104
4. `"Длинный заголовок на границе 60 символов нужен для проверки"` (62 симв) — fontSize 80
5. Искусственный title в 141 символ — проверить обрезку с `…`

### 6.2 Edge-cases

- Title из одного слова
- Title со спецсимволами (тире, кавычки внутри, emoji — должен отказаться)
- Title только латиница
- Title только кириллица
- Shrift fallback: если Literata не загрузилась — явная ошибка, не silent fallback на sans-serif

### 6.3 Автоматизация

Опционально: `tools/render-cover.test.ts` — юнит-тест только на логику скейлинга (`computeFontSize(length: number)`), без фактического рендера. Рендер PNG тестируется руками.

## 8. Что явно не делаем (YAGNI)

- Никакой админки, UI для редактирования стиля. Меняется — правится код скила.
- Никакого варьирования макета. Один жёсткий шаблон.
- Никаких фоновых иллюстраций, паттернов, градиентов.
- Никаких дополнительных шрифтов (Inter, JetBrains Mono) — только Literata для title.
- Никакого dark theme. Только светлая версия.
- Никакого автовыбора изображений из содержимого поста.
- Никакой публикации в TG из скила — скил выдаёт PNG, дальше пользователь сам.

## 9. Открытые вопросы (на будущее, не блокируют имплементацию)

- Нужен ли тёмный вариант картинки для постов о ночных релизах или про production-инциденты? — Отложено до появления реального запроса.
- Стоит ли вводить второй layout для постов-списков (где title может быть длиннее)? — Отложено до появления 3+ таких постов, где текущий шаблон визуально ломается.
- Автозапуск через hook после написания поста? — Решили нет, остаётся явный опт-ин.
